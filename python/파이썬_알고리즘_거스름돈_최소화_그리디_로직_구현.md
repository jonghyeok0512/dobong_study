### 3. 거스름돈 최소화: 그리디(Greedy)
# 거스름돈 최소화 - 가장 큰 것부터 공략하기

## 학습 포인트
-  큰 단위부터 거슬러줘야 최소 개수가 나옴.
- //(몫)으로 개수 찾고, %(나머지)로 남은 돈 업데이트하는 흐름이 중요.

## 풀이 과정
```python
def solution(change):
    answer = 0
    # 큰 동전부터 순서대로 리스트업
    coins = [500, 100, 50, 10, 5, 1]
    
    for coin in coins:
        # 이 동전으로 몇 개 줄 수 있는지 확인
        answer += change // coin
        # 거슬러 준 만큼 빼고 남은 돈으로 업데이트
        change %= coin
        
    return answer
```
## 실수 리뷰
처음엔 동전마다 개별 변수를 만들었으나, 반복문으로 리팩토링하면서 코드의 확장성이 얼마나 중요한지 깨달음. 데이터(동전 리스트)와 로직(for문)을 분리하는 연습이 더 필요함.

## 이번 학습으로 얻은 내용
성능 리뷰: O(K). 동전 종류만큼만 돌기 때문에 극강의 효율성을 가짐.

실무 관점: 자판기 시스템이나 배낭 문제(Knapsack)의 기초 버전으로, 자원 배분 최적화 로직에 응용됨.
