# 프롬프트 엔지니어링 핵심 정리

### 1. 오늘 학습 요약
프롬프트 엔지니어링은 단순히 AI에게 질문을 던지는 것이 아니라, 거대 언어 모델(LLM)이 최적의 결과물을 내놓을 수 있도록 입력값(Prompt)을 설계하고 최적화하는 기술이다. AI와의 소통에서 발생할 수 있는 모호함을 제거하고 의도를 정확히 전달하는 것이 핵심이다.

### 2. 배운 개념 정리
- **프롬프트의 4대 구성 요소**
    - **지시(Instruction)**: 모델이 수행할 구체적인 작업(요약, 번역, 코드 생성 등).
    - **문맥(Context)**: 모델이 참고해야 할 배경 정보나 가이드라인.
    - **입력 데이터(Input Data)**: 처리가 필요한 실제 본문이나 데이터.
    - **출력 지표(Output Indicator)**: 원하는 응답의 형식이나 스타일(JSON, 표, 리스트 등).

- **주요 프롬프트 기법**
    - **Zero-shot**: 예시 없이 바로 지시만 내리는 방식.
    - **Few-shot**: 모델에게 몇 가지 예시를 제공하여 패턴을 학습시키는 방식.
    - **Chain-of-Thought (CoT)**: "단계별로 생각해보자"와 같은 문구를 통해 논리적 추론 과정을 유도하는 방식.

- **구조화 및 제어 기술**
    - **구분자(Delimiters) 사용**: ###, """, --- 등을 사용하여 지시문과 데이터를 명확히 분리.
    - **역할 부여(Persona)**: "너는 10년 차 파이썬 전문가야"와 같이 페르소나를 설정하여 전문성 강화.
    - **부정적 지시보다는 긍정적 지시**: "~하지 마"보다 "~해줘"라고 명확하게 할 일을 지정하는 것이 효과적임.

### 3. 프롬프트 리뷰 (구조화 사례)
```markdown
# 나쁜 프롬프트 사례
파이썬 코드를 알려줘.

# 좋은 프롬프트 사례 (수업 내용 적용)
역할: 너는 데이터 분석 전문가야.
지시: 아래 제공된 공공데이터(JSON)에서 'pm25Value'가 20 이상인 항목만 필터링하는 파이썬 코드를 작성해줘.
조건: 
- 리스트 컴프리헨션을 사용할 것.
- 각 라인에 한글 주석을 달 것.
- 결과물은 코드 블록 안에 담아줘.
데이터: """[입력 데이터]"""
```

### 4. 실수/에러와 해결 과정
- **문제**: 지시가 모호할 때 AI가 엉뚱한 답변(환각 현상)을 내놓음.
- **해결**: '명확성'과 '구체성'이 가장 중요함을 인지하고, 출력 형식을 JSON이나 Markdown 등으로 지정하여 응답의 일관성을 확보함. 구분자를 사용하여 데이터와 명령어를 분리하는 습관을 들임.

### 5. 실무 관점의 참견 적용해보기
- **비용 최적화**: 너무 긴 프롬프트(Few-shot이 너무 많을 때)는 API 토큰 비용을 상승시킨다. 최소한의 예시로 최대의 효과를 내는 '최적의 프롬프트'를 찾는 과정이 필요하다.
- **재사용성**: 한 번 잘 만든 프롬프트는 템플릿화하여 팀 내에서 공유하거나 시스템 프롬프트로 고정하여 서비스의 품질을 유지해야 한다.
