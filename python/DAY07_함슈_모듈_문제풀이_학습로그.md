## 2. 함수(Function)에 대한 이해

### 함수의 역할
- 반복되는 코드를 하나의 이름으로 묶는다.
- 코드의 가독성과 재사용성을 높인다.
- 기능 단위로 책임을 분리할 수 있다.

### 핵심 개념
- 함수는 정의(def)와 호출(call)로 나뉜다.
- 매개변수(parameter)는 함수가 외부에서 값을 받기 위한 통로다.
- return은 함수의 실행 결과를 돌려준다.

### 정리 포인트
- 함수 내부의 변수는 함수 밖에 영향을 주지 않는다.
- 같은 코드를 여러 번 사용할 경우 함수로 만드는 것이 유리하다.

---

## 3. 모듈(Module)에 대한 이해

### 모듈이 필요한 이유
- 관련 있는 함수와 변수들을 파일 단위로 정리하기 위함
- 코드의 규모가 커질수록 파일 분리가 필수적이다.

### import 방식 정리
- import module
- from module import function
- from module import *

### 주의할 점
- from module import * 은 편하지만 가독성이 떨어질 수 있다.
- 필요한 것만 명확히 import 하는 습관이 중요하다.

## 4. 문제 풀이(problem09)을 통해 배운 점

### 문제 접근 방식
- 문제를 바로 코드로 옮기지 않고,
  입력값과 출력값 구조를 먼저 파악했다.
- 조건문과 반복문의 역할을 분리해서 생각했다.

### 핵심 포인트
- while문은 종료 조건이 명확할 때 사용한다.
- if문은 상태에 따라 동작을 나누는 역할을 한다.
- 함수로 감싸면 테스트와 재사용이 쉬워진다.
