# 랜덤 조 편성 함수 구현 학습 로그

## 1. 오늘의 문제 정의

이름이 담긴 리스트를 입력받아  
- 순서를 랜덤으로 섞고  
- 조당 인원 수를 기준으로 조를 나누며  
- 사람이 보기 좋은 형태로 출력하는  
랜덤 조 편성 함수를 구현하는 것이 목표였다.

추가 조건은 다음과 같다.

- 원본 리스트는 변경하지 않는다.
- 조당 인원 수는 기본값 4명으로 설정한다.
- 사용자가 인원 수를 입력하면 해당 값으로 조를 구성한다.
- 마지막 조 인원이 부족해도 에러 없이 처리한다.

---

## 2. 최종 완성 코드

```python
from random import shuffle

def make_group(names, size=4):
    names = names[:]      # 원본 리스트 보호
    shuffle(names)        # 전체 랜덤 섞기
    lst = {}
    group_number = 1

    for i in range(0, len(names), size):
        members = names[i:i+size]
        lst[str(group_number) + "조"] = members
        print(
            str(group_number) + "조 :",
            ", ".join(members),
            "(총 " + str(len(members)) + "명)"
        )
        group_number += 1

    return lst
```

---

## 3. 내가 넣은 기능 정리

- random.shuffle()를 이용한 전체 명단 랜덤 섞기
- names[:] 슬라이싱을 이용한 원본 리스트 보호
- size=4 기본값 매개변수를 사용한 조당 인원 옵션
- range(start, end, step)을 활용한 그룹 단위 반복
- 리스트 슬라이싱으로 조 단위 멤버 분리
- join()을 이용한 가독성 높은 출력
- 딕셔너리 형태로 조 결과 구조화
- 각 조의 총 인원 수 함께 출력

---

## 4. 만들면서 실수했던 부분과 어려웠던 점

- shuffle() 함수가 값을 반환하지 않는다는 점을 처음에 이해하지 못했다.
- 함수 내부에서 리스트를 수정하면 외부 원본 리스트도 변경되는 문제를 겪었다.
- 조당 인원 수를 size로 받았지만 코드 내부에서 숫자를 직접 사용해 옵션이 동작하지 않았다.
- names 전체와 names[i:i+size]를 구분하지 못해 출력이 잘못되었다.
- print 함수에서 쉼표 출력 시 자동 공백이 생기는 부분이 헷갈렸다.

## 5. 헷갈릴 수 있는 개념 정리

- shuffle vs sample  
  - shuffle: 전체 리스트 순서를 섞을 때 사용  
  - sample: 일부 요소만 랜덤 추출할 때 사용

- print vs return  
  - print: 화면에 출력  
  - return: 결과를 다른 코드에서 활용

- 하드코딩 값 vs 매개변수  
  - 변경 가능성이 있는 값은 매개변수로 처리

- names vs members  
  - names: 전체 명단  
  - members: 현재 조에 해당하는 명단

## 6. 느낀 점

단순한 조 편성 문제였지만,  
함수 설계에서 원본 데이터 보호와 옵션 설계의 중요성을 체감했다.

기능을 추가할 때 코드를 늘리는 방식이 아니라  
변수 하나로 전체 흐름을 제어하는 방식이 유지보수에 유리하다는 점을 배웠다.
