# [2026-01-20] Python 리스트 정렬 함수 sort()와 sorted() 차이 학습

## 1. 오늘 학습 요약
데이터 정렬 시 사용되는 `sort()` 메서드와 `sorted()` 내장 함수의 작동 원리 및 차이점을 파악함.

## 2. 배운 개념 정리
### list.sort()
- **정의**: 리스트 객체 자체를 정렬하는 메서드.
- **왜 중요한가**: 원본 데이터를 직접 수정하므로 메모리 사용량이 적음 (In-place 정렬).
- **주의점**: 반환값이 `None`이므로 `a = a.sort()`와 같이 사용하면 변수가 비어버리는 실수를 조심해야 함.

### sorted(iterable)
- **정의**: 입력된 데이터를 정렬하여 새로운 리스트를 반환하는 함수.
- **왜 중요한가**: 원본 데이터를 유지해야 하는 분석 환경이나 실무 로직에서 안전함.
- **주의점**: 새로운 리스트 객체를 생성하므로 데이터가 매우 클 경우 메모리 점유율을 고려해야 함.

## 3. 오늘 작성한 코드리뷰
```python
# [sort() 예시]
numbers = [3, 1, 4]
numbers.sort()  # numbers 자체가 [1, 3, 4]로 변함
print(numbers)

# [sorted() 예시]
origin_list = [3, 1, 4]
new_list = sorted(origin_list)  # origin_list는 그대로 [3, 1, 4]
# new_list에만 [1, 3, 4]가 담김
```

## 4. 실수/에러와 해결 과정
- **실수했던 점**: `result = data.sort()`라고 썼는데 `result`에 아무것도 안 담겨서 당황함.
- **해결**: `.sort()`는 원본만 바꾸고 결과를 반환하지 않는다는 점을 이해함. 결과값을 쓰고 싶다면 `sorted()`를 써야 함.

## 5. 실무 관점의 참견
- 데이터 원본은 '불변(Immutable)'하게 관리하는 것이 버그를 줄이는 지름길임. 특별히 메모리가 부족한 환경이 아니라면 `sorted()` 사용을 권장함.
