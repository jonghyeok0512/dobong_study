#### 1. 오늘 학습 요약
- 클래스 간의 관계를 설정하는 **Composition(합성)**의 개념을 배우고, 이를 스마트홈 시스템의 프로토콜 연결 로직에 실무적으로 적용해 봄.

#### 2. 배운 개념 정리
- **Composition(합성)**: 'Has-a' 관계. 한 클래스가 다른 클래스의 인스턴스를 자신의 속성으로 가지는 것. 상속보다 결합도가 낮아 유지보수에 유리함.
- **동적 교체**: 객체 생성 시점에 필요한 부품(객체)을 주입하여 기능을 자유롭게 바꾸는 방식.
- **상속 vs 합성**: 상속은 부모의 특징을 물려받는 것이고, 합성은 필요한 기능을 가진 객체를 조립하여 사용하는 것임.

#### 3. 오늘 작성한 코드리뷰 (상세 주석)
```python
# [Composition의 핵심 예시]
class Smarthub:
    def __init__(self, name, protocol):
        self.name = name
        # Protocol 객체를 인자로 받아 내부 속성에 저장 (합성)
        self.protocol = protocol  
        self.devices = []

    def activate_all(self):
        for device in self.devices:
            # 합성된 내부 객체(protocol)의 메서드를 연쇄적으로 호출
            # '객체.속성(객체).메서드()' 흐름을 파악하는 것이 중요
            device.protocol.start_connection()
            device.turn_on()
```

#### 4. 실수/에러와 해결 과정 (사고 훈련)
- **에러 상황**: 터미널에서 `git commit -m '...` 입력 중 따옴표를 닫지 않아 `>` 표시가 뜨며 명령어가 종료되지 않음.
- **사고 과정**: 에러 메시지가 없는 이유는 컴퓨터가 명령이 아직 끝나지 않았다고 판단했기 때문임을 파악.
- **해결**: `Ctrl + C`를 눌러 강제 종료하거나, 열려 있는 따옴표(`'`)를 마저 입력하여 문장을 종결시킴.

#### 5. 성능 및 시간 복잡도 리뷰
- **합성의 효율성**: 상속처럼 깊은 계층 구조를 만들지 않으므로 메모리 탐색 효율이 좋음.
- **연쇄 호출 성능**: `device.protocol.start_connection()`은 직접 참조 방식이므로 시간 복잡도 $O(1)$로 매우 빠르게 동작함.

#### 6. 실무 / RAG 관점의 참견
- **실무 적용**: 오늘 배운 합성은 '전략 패턴(Strategy Pattern)'의 기초가 됨. 나중에 AI 모델을 바꿀 때(예: GPT-4에서 Claude 3로) 코드 전체를 고치지 않고 모델 객체만 갈아 끼우는 식으로 응용 가능함.
- **RAG 관점**: 검색기(Retriever) 객체를 합성으로 관리하면, 상황에 따라 다른 검색 알고리즘을 유연하게 교체하는 시스템을 구축할 수 있음.
