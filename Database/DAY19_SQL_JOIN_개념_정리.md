# 오늘 학습 요약 (JOIN 핵심 정리)
두 개 이상의 테이블을 특정 기준(결합 조건)에 따라 연결하여 하나의 결과 집합으로 만드는 JOIN에 대해 학습함.
데이터가 물리적으로 분리되어 있어도 JOIN을 통해 비즈니스 로직에 필요한 통합 데이터를 조회할 수 있음.

---

# 배운 개념 정리
1. **INNER JOIN (내부 조인)**
   - 정의: 양쪽 테이블에 조인 조건이 일치하는 데이터만 가져옴.
   - 중요성: 가장 많이 사용되며, 교집합 데이터만 필요할 때 필수적임.
   - 주의점: 한쪽에만 데이터가 있으면 결과에서 누락됨.

2. **LEFT (OUTER) JOIN (왼쪽 외부 조인)**
   - 정의: 왼쪽 테이블의 모든 데이터와 오른쪽 테이블의 일치하는 데이터를 가져옴.
   - 중요성: 기준이 되는 왼쪽 테이블의 데이터 유실 없이 정보를 붙일 때 사용함.
   - 주의점: 오른쪽 테이블에 매칭되는 데이터가 없으면 NULL로 채워짐.

3. **RIGHT (OUTER) JOIN (오른쪽 외부 조인)**
   - 정의: 오른쪽 테이블의 모든 데이터와 왼쪽 테이블의 일치하는 데이터를 가져옴.
   - 중요성: LEFT JOIN과 방향만 반대이며, 보통 LEFT JOIN을 선호함.

4. **FULL (OUTER) JOIN (완전 외부 조인)**
   - 정의: 양쪽 테이블의 모든 데이터를 가져오며, 매칭되지 않는 부분은 NULL 처리함.
   - 중요성: 양측의 모든 유실 데이터를 확인해야 할 때 사용함.

# 오늘 작성한 코드리뷰
```sql
/* [미션] 영화 재고(inventory)와 영화 제목(film)을 결합하여 
어떤 매장에 어떤 영화가 있는지 조회하기
*/

SELECT 
    i.inventory_id,  -- 재고 고유 ID
    f.title,        -- 영화 제목 (film 테이블에서 가져옴)
    i.store_id      -- 보유 매장 ID
FROM inventory i
-- INNER JOIN 사용: 재고가 실존하는 영화만 매칭 (재고 없는 영화 제외)
INNER JOIN film f ON i.film_id = f.film_id;

/*
주의사항: 
1. 테이블 별칭(i, f)을 사용하여 컬럼 소속을 명확히 해야 성능과 가독성이 좋아짐.
2. ON 절의 조건 컬럼이 인덱스로 설정되어 있는지 확인하는 것이 실무에서 중요함.
*/
